# Claude Code в Telegram

*Read this in [English](README.md).*

Мост между Telegram и Claude Code, запущенным на твоих серверах. Сессии общие
с VS Code: что начато в редакторе через Remote SSH, можно продолжить в
Telegram, и наоборот. Бот не хранит отдельную копию контекста, он работает
поверх тех же `~/.claude/projects/*.jsonl` на сервере.

**Модель работы:** настройка — в личном чате с ботом, а сам диалог с Claude —
в отдельной рабочей группе, где каждая сессия открывается своей темой со своей
историей. В личке Claude не отвечает — она только панель управления.

## Возможности

- Несколько пользователей, у каждого свой список машин (SSH-доступ).
- Добавление машины прямо в чате; SSH-ключи шифруются (Fernet) и проверяются
  при добавлении.
- **Установка Claude Code на сервер прямо из бота** — проверяет Node.js
  (ставит через nvm/fnm при отсутствии), затем ставит
  `@anthropic-ai/claude-code` в user-local npm prefix (без root), с живым
  прогрессом.
- Подключение рабочей группы в один тап (deep-link `?startgroup`) с
  автопривязкой: бот сам цепляет группу, как только его сделали админом в
  супергруппе с темами. Команда `/bindgroup` остаётся как ручной запасной путь.
- Каждая сессия = отдельная тема в группе. Бот сам создаёт темы.
- Продолжение существующих сессий (`claude --resume`) с подгрузкой последних
  сообщений (recap) в тему при открытии. Создание новых сессий.
- Live-стриминг работы Claude (`--output-format stream-json`): видны вызовы
  инструментов и текст по мере генерации, с таймером во время долгих тихих шагов.
- Выбор модели (opus / sonnet / haiku) на тему командой `/model`.
- Отправка файлов и фото: загружаются на сервер, путь передаётся Claude.
- Кнопка «Остановить» и команда `/stop`.
- Восстановление авторизации Claude на сервере через бота (см. ниже).
- Whitelist по Telegram ID.

## Почему именно группа с темами

Telegram не даёт боту переписать историю личного чата под выбранную сессию
(старые сообщения нельзя задним числом подменить). Поэтому «отдельная история
на каждую сессию» делается темами форума: открыл сессию — бот создал тему,
подтянул recap и навсегда привязал её к этой сессии. Переключение между
сессиями — это просто переход между темами.

> Бот **не может создать группу сам** — Bot API такого не умеет, группы создают
> только пользовательские аккаунты. Но всё остальное автоматизировано: кнопка
> добавления бота в группу + автопривязка при выдаче прав. Один раз создаёшь
> группу руками, дальше темы появляются сами.

## Требования на серверах

На каждой машине нужен Claude Code CLI. Можно поставить его из бота (см.
«Возможности»), либо проверить существующую установку:

```bash
claude --version   # должен отвечать
```

Бот вызывает `claude` через `bash -lc`, так что PATH из
`~/.bashrc`/`~/.profile`/nvm подхватывается. Permission-режим —
`bypassPermissions` (как Bypass permissions в VS Code).

### Авторизация Claude и её восстановление через бота

Claude на сервере должен быть залогинен по подписке (`claude auth`). Если
сервер «потерял» логин (разлогинился, протух токен, другой аккаунт), не нужно
лезть на машину руками:

1. На своей машине с браузером выполни `claude setup-token` — это выдаёт
   долгоживущий (≈год) токен подписки.
2. В личном чате с ботом: `/machines` → значок 🔑/🔓 рядом с машиной → «Войти по
   подписке» → пришли токен.
3. Бот шифрует токен, кладёт его на сервер файлом `~/.claude/.tg-anthropic-key`
   с правами `600` (через SFTP, не через командную строку — токен не светится в
   `ps`), и экспортирует как `CLAUDE_CODE_OAUTH_TOKEN`. Каждый запуск Claude
   подхватывает его, не трогая обычный `claude auth`.
- «Убрать авторизацию» — удаляет файл с сервера и запись из базы.

Это именно **подписочная** авторизация (оплата по Pro/Max), а не API-биллинг по
токенам. Если запуск падает с ошибкой авторизации, бот сам подскажет восстановить
вход. Сообщение с токеном удаляется из чата сразу после обработки.

## Запуск

Нужен Docker с плагином compose. Бот работает на long polling: входящие порты
не нужны, только исходящий доступ к `api.telegram.org`.

```bash
git clone https://github.com/OneSilverFinger/claude-tg-bot.git
cd claude-tg-bot
cp .env.example .env
# заполни BOT_TOKEN, ALLOWED_USER_IDS, MASTER_KEY (генератор внутри файла)
docker compose up -d --build
docker compose logs -f
```

### Откуда взять значения для .env

- `BOT_TOKEN` — у @BotFather (`/newbot`). После создания включи боту доступ к
  группам: @BotFather → бот → Group Privacy → Turn off (иначе он не увидит
  сообщения в темах).
- `ALLOWED_USER_IDS` — свой Telegram ID узнай у @userinfobot. Через запятую,
  если пользователей несколько.
- `MASTER_KEY` — сгенерируй один раз и больше не меняй:
  ```bash
  python3 -c "import base64,os;print(base64.urlsafe_b64encode(os.urandom(32)).decode())"
  ```

Данные (SQLite с машинами и привязками) лежат в docker-volume `bot-data` и
переживают пересборку. Обновление: `git pull && docker compose up -d --build`.

## Первое использование

1. Напиши боту `/start` в личном чате.
2. «Машины» → «Добавить машину»: имя, хост, порт, логин, ключ/пароль. Ключ
   пришли файлом или текстом — сообщение удаляется сразу после обработки, ключ
   шифруется. Бот тут же проверит подключение и наличие `claude` (и предложит
   установить, если его нет).
   - Если Claude на сервере не авторизован, задай токен подписки (значок 🔑 →
     «Войти по подписке», см. раздел про авторизацию выше).
3. «Подключить рабочую группу» → добавь бота в группу (или создай новую в том
   же диалоге), включи в ней «Темы» и сделай бота админом. Бот привяжет группу
   автоматически и напишет об этом.
4. «Проекты и сессии» → выбери проект → выбери сессию (или «Новая сессия»). Бот
   откроет её отдельной темой в группе.
5. Перейди в эту тему и общайся с Claude. Файлы и фото кидай прямо в тему.

### Команды внутри темы

- `/sessions` — сменить проект/сессию этой темы.
- `/model` — выбрать модель для этой темы.
- `/status` — что сейчас привязано к теме.
- `/stop` — остановить текущий запуск Claude.
- `/unbind` — отвязать тему от сессии (сама сессия на сервере не трогается).

## Безопасность

- Доступ только из whitelist (`ALLOWED_USER_IDS`). Чужие сообщения
  игнорируются, человеку показывается его ID для запроса доступа.
- SSH-секреты шифруются Fernet-ключом из `MASTER_KEY`, в базе лежит только
  шифртекст. Потеря `MASTER_KEY` = нужно заново добавить машины.
- Сообщения с ключами/паролями удаляются из чата сразу после обработки.
- `bypassPermissions` означает, что Claude выполняет команды на сервере без
  подтверждений. Давай доступ к боту только доверенным людям и используй
  отдельные SSH-аккаунты с нужным уровнем прав.

## Архитектура

```
bot/
  config.py            загрузка и валидация .env
  crypto.py            шифрование SSH-секретов (Fernet)
  db.py                SQLite: машины, привязки чат→сессия, префы
  access.py            middleware whitelist
  ssh.py               пул SSH-подключений (asyncssh), тест машины
  claude.py            запуск claude, парсинг stream-json, списки сессий, recap, установка
  render.py            stream-json → сообщения, live-edit, markdown→HTML, чанки
  keyboards.py         inline-клавиатуры
  handlers_menu.py     /start, /menu, модель, статус, /bindgroup
  handlers_machines.py добавление/выбор/удаление машин (FSM), установка Claude
  handlers_sessions.py проекты, сессии, recap, открытие темой
  handlers_chat.py     основной диалог, файлы, /stop
  main.py              сборка приложения и polling
```

## Известные ограничения

- Бот не создаёт группу сам (ограничение Bot API) — группу один раз создаёшь
  руками, дальше темы автоматические.
- Один активный запуск Claude на тему за раз (защита от гонок в сессии).
- Не пишите в одну сессию одновременно из VS Code и из бота.
- Recap показывает последние несколько сообщений, не всю историю (она может
  быть огромной). Полный контекст всё равно у Claude через `--resume`.
- Автопривязка группы срабатывает на событии прав бота. Если «Темы» включили
  уже после выдачи прав, событие не повторяется — просто отправь `/bindgroup`
  в группе.
- Permission-кнопки в чате (Разрешить/Запретить) — кандидат на v2; сейчас
  режим bypass.
